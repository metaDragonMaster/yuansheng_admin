### 接口命名规范
```
  接口函数名体
  开头 模块 结尾

  select 开头   查询，用于列表、对象信息的查询 通常结尾为List或Info
  add    开头   添加，用于添加数据
  update 开头   修改，用于修改数据
  delete 开头   删除、移除， 用于对作用对象数据的移除
  set    开头   设置，本质上也是修改，用于对某个数据状态的更改。
  get    开头   获取，本质上也是查询，用于获取某个数据。
  ById   结尾   接口用了某个对象的 ID或者接口加上ID值  数据类型"1"/1,接口上"/api/${id}"
  ByIds  结尾   接口用了多个对象的 ID或者接口加上ID值。数据类型"1,2,3"，接口上"/api/${ids}"
```

### 接口类型 
```
  post    用于添加数据
  get     用于获取数据
  put     用于修改数据
  delete  用于删除数据
```
